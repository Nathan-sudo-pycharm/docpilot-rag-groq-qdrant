# DocPilot v1.1 — Source Citations Feature Spec

## What it is
After the LLM streams its answer, each assistant message shows 
source lines like:

    Source: faq.pdf · page 3
    Source: manual.pdf · page 7

Makes the system transparent — users can see exactly which chunk 
of which document generated the answer.

## Why it was deferred
Requires a real backend change. The `/chat` endpoint currently 
only streams answer text. Citations need a separate SSE event 
carrying chunk metadata (filename + page number). Kept out of 
the core build to ship a clean MVP first.

## Release plan
1. Tag v1.0.0 (core build done)
2. Build citations ~2 days later as dedicated v1.1
3. Record LinkedIn demo GIF after citations land — retrieval 
   transparency makes it a more impressive demo

## What v1.1 needs

### Backend — `backend/app/services/rag.py`
`stream_answer()` yields source metadata after the answer text 
as a separate SSE event:
    data: {"sources": [{"filename": "faq.pdf", "page": 3}]}
    data: [DONE]

### Frontend — `frontend/src/hooks/useChat.ts`
Parse the new event type and attach sources to the assistant 
message object alongside the text content.

### Frontend — `frontend/src/components/MessageBubble.tsx`
Render `Source: file.pdf · page N` lines in muted small text 
below the answer bubble.
